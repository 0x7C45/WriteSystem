## 一、Agent 元信息

| 字段 | 说明 | 示例值 |
|------|------|--------|
| **name** | Agent 标识符（小写字母+连字符） | `paper-order-analyst` |
| **version** | 语义化版本号 | `1.0.0` |
| **type** | Agent 类型 | `executor` / `reviewer` / `advisor` |
| **phase** | 所属管线阶段 | `0` / `1a` / `1b` / `2` / `3` / `3.5` |
| **depends_on** | 前置依赖 Agent | `["paper-literature-agent"]` |
| **tools** | 使用的工具列表 | `["search_blocks", "validate_citations"]` |
| **outputs** | 产出的文件列表 | `["00_订单信息/订单摘要.md"]` |

### 字段详细说明

#### `name`
- **格式**: `{domain}-{role}`，全小写，单词间用连字符
- **命名约定**:
  - 论文管线 Agent: `paper-{role}`
  - 审查 Agent: `paper-{aspect}-reviewer`
  - 通用 Agent: `{domain}-{role}`

#### `version`
- **格式**: `major.minor.patch`（语义化版本）
- **变更规则**:
  - `major`: 不兼容的 API 变更（输入输出格式变化）
  - `minor`: 向后兼容的功能新增（新增工具、新增检查项）
  - `patch`: 向后兼容的问题修复（修正 bug、文档更新）

#### `type`
- **executor**: 执行具体工作任务（撰写、分析、生成）
- **reviewer**: 审查质量（逻辑、数据、格式）
- **advisor**: 里程碑决策（唯一批准权）

#### `phase`
- 指定 Agent 在管线中的执行阶段
- 可选值: `0`, `1a`, `1b`, `2`, `3`, `3.5`
- 跨阶段 Agent 可写数组: `["1a", "1b", "2"]`

#### `depends_on`
- 列出必须先执行完成的 Agent
- 用于管线调度和并行控制
- 空数组 `[]` 表示无依赖（可立即启动）

#### `tools`
- 列出 Agent 使用的工具名称
- 工具定义在 `tools/mcp-servers/` 或 `tools/skills/`
- 用于权限检查和工具可用性验证

#### `outputs`
- 列出 Agent 产出的文件路径（相对于项目根目录）
- 用于产物追踪和依赖检查
- 文件路径使用项目标准目录结构（00-07）
