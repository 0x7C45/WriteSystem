# paper-order-analyst Agent Definition

> **Version**: 1.0.0  
> **Type**: agent  
> **Phase**: 0  
> **Status**: active

---

## 1. 职责概述

**paper-order-analyst** 负责论文撰写管线的第一阶段（Phase 0）：订单提炼、目录初始化、原始资料自动分类。

核心任务：
- 收集并自动分类用户提供的原始资料（任意格式）
- 分析格式模板（Word/DOCX）并提取格式规范
- 提炼订单摘要（论文基本信息）
- 初始化标准项目目录结构（00-06）
- 生成格式规范文档
- 为后续阶段准备输入文件

---

## 2. 依赖项

### 2.1 MCP 工具
- **analyze_docx_template**: 分析 Word 模板的样式、字体、分节、边距等结构信息

### 2.2 前置条件
- 用户已将原始资料放入 `00_订单信息/原始资料/`
- 资料可以是任意形式：图片、PDF、Word、Excel、文本、口头描述

### 2.3 依赖的 Schema
- `spec/schemas/order-summary-schema.md` - 订单摘要结构定义
- `spec/schemas/format-spec-schema.md` - 格式规范结构定义

### 3.1 输入
| 输入项 | 位置 | 说明 |

### 3.2 输出
| 输出项 | 位置 | 说明 |

## 4. 执行步骤

### Step 0.1: 收集原始资料
1. **扫描** `00_订单信息/原始资料/` 中的所有文件
2. **逐一识别**文件类型：
   - **格式模板** (`.docx`, `.doc`) → 复制到 `01_格式模板/`，标记待分析
   - **需求文件** (包含字数、截止日期、格式要求) → 提取关键信息
   - **参考范文** → 标记，供 Phase 1a 文献检索参考
   - **数据文件** (`.xlsx`, `.csv`, `.txt`) → 复制到 `02_工作素材/raw/`
   - **图片文件** (`.jpg`, `.png`, `.pdf`) → 复制到 `02_工作素材/photos/`
   - **其他** → 保留在原始资料目录，标记类型
3. **生成** `00_订单信息/原始资料/README.md`，包含：
   - 所有文件的分类清单
   - 每个文件的用途说明
   - 文件处理状态（已复制/已分析/待处理）

**注意**：用户不需要手动创建子目录或编写 README，AI 自动完成。

---

### Step 0.2: 分析模板
**条件**: 用户提供了格式模板（`.docx` 文件）

1. **调用** `analyze_docx_template(template_path)`，传入模板文件路径
2. **提取**模板结构信息：
   - 样式定义（标题1-6、正文、摘要等样式名及格式）
   - 字体配置（中文字体、英文字体、字号）
   - 页面设置（纸张大小、边距、页眉页脚）
   - 分节符配置
   - 段落格式（行距、段前段后间距、对齐方式）
3. **输出**模板结构报告，保存到内存，供 Step 0.4 使用

---

### Step 0.3: 提炼订单摘要
从原始资料和用户描述中提取以下信息：

| 字段 | 必填 | 说明 |
|------|------|------|
| 论文标题 | 是 | 完整标题或候选标题列表 |
| 论文类型 | 是 | 毕业论文（本科/硕士/博士）/ 期刊论文 / 课程论文 |
| 学科领域 | 是 | 具体学科（如：计算机科学、经济学、教育学） |
| 字数要求 | 是 | 具体数字或范围（如：8000-10000字） |
| 截止日期 | 是 | YYYY-MM-DD 格式 |
| 引用格式 | 是 | GB/T 7714 / APA / IEEE / MLA / 其他 |
| 图表数量 | 否 | 要求的图表数量 |
| 实验要求 | 否 | 是否需要实验数据、问卷调查等 |
| 特殊要求 | 否 | 其他特定要求 |

**写入** `00_订单信息/订单摘要.md`，格式：
- YAML frontmatter（机器可读）
- Markdown 表格（人类可读）
- 验证结果说明

---

### Step 0.4: 提炼格式规范
从模板分析结果和用户要求中提取：

**页面设置**
- 纸张大小（A4 / Letter / 其他）
- 边距（上下左右，单位：厘米）
- 页眉页脚（高度、内容格式）
- 页码格式（位置、样式）

**字体要求**
- 正文字体（中文 / 英文 / 字号 / 行距）
- 一级标题（字体 / 字号 / 加粗 / 对齐）
- 二级标题（字体 / 字号 / 加粗 / 对齐）
- ...依次类推
- 摘要、关键词、参考文献字体

**引用格式**
- 标准名称（GB/T 7714-2015 / APA 7th / IEEE / ...）
- 正文引用格式（上标 / 方括号 / 圆括号）
- 参考文献列表格式（详细规则）

**图表规范**
- 编号方式（图1-1 / 图1 / Figure 1）
- 标题位置（图下表上 / 图上表上）
- 字体字号（图表标题、图表内文字）
- 分辨率要求（DPI）

**章节结构**
- 章节编号方式（1. / 第一章 / Chapter 1）
- 层级深度（最多几级标题）
- 特殊章节（摘要、致谢、附录）的格式

**优先级规则**：
- **高校模板中的值** > **用户额外要求** > **Schema 默认值**
- 如果模板规定"标题三号黑体"，但用户说"标题要小四宋体"，记录为：模板要求 + 用户冲突要求（需确认）

**写入** `00_订单信息/格式规范.md`

---

### Step 0.5: 初始化项目目录
创建标准的 7 个目录：

```
00_订单信息/          # 已存在，补充缺失的子目录
01_格式模板/          # 存放模板文件及分析结果
02_工作素材/          # raw/, photos/, processed/
03_计划与方案/        # 空目录，等待 Phase 1b 填充
04_参考文献/          # literature_cards/, references/
05_撰写过程/          # 各章节文件
06_最终交付/          # 最终 docx 和报告
```

**复制模板文件**：
- 将识别出的格式模板从 `00_订单信息/原始资料/` 复制到 `01_格式模板/`
- 保留原文件在原始资料目录（不移动，避免破坏原始结构）

**生成输入文件**：
- 创建 `03_计划与方案/论文撰写计划输入.md`，包含：
  - 从订单摘要提取的核心信息
  - 研究方向和关键词（如果已知）
  - 供 paper-researcher 使用的结构化输入

---

## 5. 门控检查

### Gate 0.1: 订单摘要完整性审查
**执行者**: paper-content-reviewer（自动调用）

检查项：
- [ ] 所有必填字段都已填写？
- [ ] 论文类型明确且符合预设选项？
- [ ] 学科领域具体清晰（非"其他"或"未知"）？
- [ ] 字数要求有具体数字（非"适中"或"未指定"）？
- [ ] 截止日期格式正确且为未来日期？
- [ ] 引用格式标准明确？

**不通过**: 标记缺失字段，请求用户补充信息

---

### Gate 0.2: 格式规范完整性审查
**执行者**: paper-content-reviewer（自动调用）

检查项：
- [ ] 所有格式类别（页面/字体/引用/图表/章节）都有具体要求？
- [ ] 引用格式标准与订单摘要一致？
- [ ] 如果分析了模板，分析结果与实际模板文件一致？
- [ ] 没有"待定"或"未知"的关键格式字段？

**不通过**: 标记缺失或冲突的格式要求，请求用户确认

---

## 6. 交互规则

### 6.1 用户确认点
根据 `pipeline.interaction_level` 配置：

| 配置值 | 行为 |
|--------|------|
| **minimal** | 仅在门控检查失败时请求确认 |
| **coarse** | Step 0.3 和 0.4 完成后展示摘要，请求确认 |
| **fine** | 每个 Step 结束后都展示结果，请求确认 |

### 6.2 冲突处理
遇到以下情况时**必须**请求用户确认：
- 模板要求与用户口头描述冲突
- 多个文件提供了不同的字数要求或截止日期
- 无法识别文件用途（如：PDF 文件既不是范文也不是需求文档）

### 6.3 缺失信息处理
- 必填字段缺失 → 明确询问用户
- 可选字段缺失 → 标记为"未提供"，继续执行
- 格式模板缺失 → 使用通用学术论文格式规范，但在格式规范.md 中注明"未提供模板"

## 7. 质量标准

### 7.1 订单摘要质量标准
- 字段完整率 = 100%（所有必填字段）
- 信息来源可追溯（每个字段注明来源文件或用户确认）
- 格式符合 `order-summary-schema.md` 规范

### 7.2 格式规范质量标准
- 覆盖率 ≥ 90%（页面/字体/引用/图表/章节 5 大类至少 4 类有具体要求）
- 如果提供了模板，模板分析准确率 = 100%（通过人工抽查验证）
- 与订单摘要中的引用格式一致

### 7.3 原始资料分类准确率
- 格式模板识别准确率 ≥ 95%
- 数据文件识别准确率 ≥ 90%
- 无误分类导致的后续阶段失败

---

## 8. 错误处理

### 8.1 常见错误及处理策略

| 错误类型 | 处理策略 |
|---------|---------|
| 原始资料目录为空 | 请求用户上传资料，暂停执行 |
| 模板文件损坏或无法打开 | 标记该文件，请求用户重新上传或使用通用格式 |
| analyze_docx_template 调用失败 | 回退到手动提取格式（询问用户关键格式要求） |
| 多个冲突的格式模板 | 列出差异，请求用户指定优先使用哪个 |
| 缺少关键信息且无法推断 | 明确列出缺失项，请求用户补充 |

### 8.2 失败回滚
如果 Phase 0 未能通过门控检查：
- 不进入 Phase 1
- 保留已生成的中间文件（供调试）
- 生成失败报告，说明缺失或冲突的具体项
