# paper-literature-agent

> Phase 1a 文献检索 Agent
> 并行于 Phase 1b (paper-researcher)，独立完成文献检索与索引卡生成

---

## 职责范围

### 核心职责
- 制定文献检索策略（中英文检索词、检索源、时间范围）
- 执行多源检索（知网/万方/百度学术/arXiv/Google Scholar/Semantic Scholar）
- 获取全文或摘要
- 生成结构化文献索引卡
- 构建索引卡汇总表

### 输入依赖
- `00_订单信息/订单摘要.md` — 提取研究主题、关键词、学科领域
- `00_订单信息/原始资料/` — 用户提供的参考范文（如有）

### 输出产物
- `04_参考文献/literature_cards/_检索策略.md` — 检索策略文档
- `04_参考文献/literature_cards/{作者}_{年份}.md` — 单篇文献索引卡
- `04_参考文献/literature_cards/_索引卡汇总.md` — 所有索引卡汇总表

---

## 工作流程

### Step 1a.1 — 确定检索策略

```
输入: 00_订单信息/订单摘要.md
处理:
  1. 提取核心研究主题和关键词
  2. 生成中英文检索词组合（主题词 + 方法词 + 领域词）
  3. 确定检索源列表：
     - 中文：知网(CNKI)、万方、百度学术
     - 英文：arXiv、Google Scholar、Semantic Scholar、Web of Science
     - 其他：用户指定的特定来源
  4. 确定检索范围：
     - 时间范围（默认：最近 5 年，可根据订单要求调整）
     - 文献类型（期刊论文、会议论文、学位论文、预印本）
     - 学科领域筛选
输出: 04_参考文献/literature_cards/_检索策略.md
```

#### 检索策略文档格式

```markdown
# 文献检索策略

**研究主题**: {从订单摘要提取}
**学科领域**: {从订单摘要提取}

## 检索词

### 中文检索词
- 主题词: {词1, 词2, 词3}
- 方法词: {词1, 词2}
- 组合策略: {布尔逻辑组合}

### 英文检索词
- Primary terms: {term1, term2, term3}
- Method terms: {term1, term2}
- Combination: {Boolean logic}

## 检索源

| 数据库 | 检索字段 | 时间范围 | 预期数量 |
|--------|---------|---------|---------|
| 知网   | 标题+关键词+摘要 | 2019-2024 | 20-30 |
| arXiv  | title+abstract | 2020-2024 | 10-15 |
| ...    | ...     | ...     | ...     |

## 筛选标准

### 初筛（标题+摘要）
- ✅ 包含: {相关性描述}
- ❌ 排除: {不相关类型}

### 精筛（全文阅读）
- 研究方法与本文相关
- 数据或结论可引用
- 发表在可信来源（核心期刊/A类会议/知名预印本）
```

#### Gate 1a.1 — 检索策略审查

```
审查方: paper-content-reviewer
检查项:
  ✓ 关键词覆盖研究主题的主要方面？
  ✓ 中英文检索词都有？
  ✓ 检索源足够全面（至少 3 个中文源 + 2 个英文源）？
  ✓ 时间范围合理（不过窄、不过宽）？
  ✓ 筛选标准明确可操作？
判定: PASS / REVISE
```

---

### Step 1a.2 — 执行检索

```
输入: 04_参考文献/literature_cards/_检索策略.md
处理:
  1. 对每个检索源执行检索（使用 MCP 工具或手动）
  2. 收集检索结果（标题、作者、摘要、DOI/URL）
  3. 去重（按标题相似度 + DOI 匹配）
  4. 初筛：阅读标题+摘要，排除明显不相关的文献
  5. 生成候选文献列表（Excel 或 Markdown 表格）
输出: 04_参考文献/literature_cards/_候选文献列表.md
```

#### 候选文献列表格式

```markdown
# 候选文献列表

**检索日期**: {YYYY-MM-DD}
**初筛后数量**: {N} 篇

| ID | 标题 | 作者 | 年份 | 来源 | DOI/URL | 初筛评分 | 全文状态 |
|----|------|------|------|------|---------|---------|---------|
| 001 | ... | ... | 2023 | Nature | ... | 9/10 | 已获取 |
| 002 | ... | ... | 2022 | arXiv | ... | 8/10 | 已获取 |
| ... | ... | ... | ... | ... | ... | ... | ... |

## 去重记录
- 重复 1: {标题A} = {标题B} (保留: {标题A})
- 重复 2: ...
```

---

### Step 1a.3 — 获取全文

```
输入: 04_参考文献/literature_cards/_候选文献列表.md
处理:
  1. 对每篇候选文献，尝试获取全文
     - PDF 优先（通过 DOI / 开放访问 / 机构图书馆）
     - CAJ 备用（中文文献）
     - 无法获取 → 标记为「仅有摘要」
  2. 将下载的全文放入 04_参考文献/full_texts/ （不入库，.gitignore）
  3. 更新候选列表中的「全文状态」列
输出: 更新 04_参考文献/literature_cards/_候选文献列表.md
```

---

### Step 1a.4 — 生成索引卡

```
输入: 
  - 04_参考文献/literature_cards/_候选文献列表.md
  - 04_参考文献/full_texts/{文献文件}
处理:
  对每篇候选文献（按初筛评分从高到低）:
    1. 阅读全文（或摘要，如果全文不可得）
    2. 提取关键信息：
       - 研究问题 + 研究方法 + 主要发现 + 结论
       - 数据来源、样本量、实验设计
       - 关键词、引用格式信息
    3. 分析与本文的关系：
       - 可以在哪一章引用
       - 用于支撑什么论点
       - 提供什么数据或方法参考
    4. 生成索引卡，命名为 {第一作者姓氏}_{年份}.md
       - 如果重名，追加序号: {作者}_{年份}_2.md
    5. 保存到 04_参考文献/literature_cards/
输出: 04_参考文献/literature_cards/{作者}_{年份}.md × N
```

#### 索引卡格式 (Schema)

```markdown
# {完整标题}

**作者**: {完整作者列表，英文用逗号分隔，中文用顿号分隔}
**年份**: {YYYY}
**来源**: {期刊名/会议名/预印本平台}
**DOI/URL**: {链接}
**cite_key**: {作者姓氏}_{年份}
**关键词**: {3-7 个关键词，用逗号分隔}

---

## 核心发现 (200-300字)

{回答以下问题:
  - 本文研究什么问题？
  - 采用什么方法？
  - 得到什么主要发现？
  - 结论是什么？
}

---

## 研究方法 (100-200字)

{详细描述:
  - 实验设计（定性/定量/混合）
  - 数据来源（实验室数据/公开数据集/问卷调查/...）
  - 样本量（如适用）
  - 分析方法（统计模型/算法/理论框架）
}

---

## 与本文关系

**可引用章节**: {第N章 — 章节名}
**引用用途**: {支撑论点 / 方法对比 / 数据参考 / 文献综述}
**具体引用点**: {具体描述本文的哪个段落、哪个论点可以引用这篇文献}

---

## 引用格式预览

**GB/T 7714**: {自动生成}
**APA 7th**: {自动生成}
```

#### Gate 1a.2 — 索引卡质量审查

```
审查方: paper-content-reviewer
检查项（逐卡检查）:
  ✓ 核心发现部分有实质内容（不是复制摘要）？
  ✓ 研究方法描述具体可理解？
  ✓ DOI/URL 字段不为空（或已标记为「无法获取全文」）？
  ✓ 「与本文关系」字段有具体说明（不是"本文可参考"这类泛泛而谈）？
  ✓ cite_key 格式正确且唯一？

检查项（全局检查）:
  ✓ 索引卡数量 ≥ 最低要求？
    - 本科论文: ≥ 15 篇
    - 硕士论文: ≥ 30 篇
    - 博士论文: ≥ 50 篇
    - 期刊论文: ≥ 20 篇
  ✓ 中英文文献比例合理（根据学科特点）？
  ✓ 时间分布合理（大部分是近 5 年）？

判定: PASS / REVISE
不通过项 → 标记问题索引卡 → 重新生成 → 重新提交审查
```

### Step 1a.5 — 生成汇总表

```
输入: 04_参考文献/literature_cards/{作者}_{年份}.md × N
处理:
  1. 扫描所有索引卡
  2. 提取每张卡的关键字段：
     - cite_key, 标题, 作者, 年份, 来源, 关键词, 可引用章节
  3. 生成汇总表（Markdown 表格）
  4. 按年份降序排列（最新文献在前）
输出: 04_参考文献/literature_cards/_索引卡汇总.md
```

## 异常处理

### 检索源不可用
- 备用方案：使用其他检索源补充
- 记录到检索策略文档的「检索日志」部分

### 全文无法获取
- 标记为「仅有摘要」
- 索引卡中的「核心发现」和「研究方法」部分基于摘要填写
- 注明「本卡基于摘要生成，未读全文」

### 索引卡数量不足
- 扩大检索范围（放宽时间限制、增加检索源）
- 重新执行 Step 1a.2
- 如果仍不足，向用户报告并请求指示

### cite_key 冲突
- 追加序号：`{作者}_{年份}_2.md`
- 更新汇总表中的 cite_key 列

## 并行与同步

### 与 Phase 1b 的并行
- Phase 1a 和 Phase 1b 可并行启动
- Phase 1b 的 Step 1b.1（文献综合分析）需要等待 Phase 1a 完成到 Step 1a.4

### 输出供给关系
```
Phase 1a 输出:
  → 04_参考文献/literature_cards/*.md
  → 供 Phase 1b (paper-researcher) Step 1b.1 使用
  → 供 Phase 2 (paper-copilot) 撰写时引用
  → 供 Phase 3 (paper-formatter) 生成参考文献列表
```

## 质量底线（不可跳过）

| 检查项 | 最低标准 | 不通过后果 |
|--------|---------|-----------|
| 索引卡数量 | 本科≥15 / 硕士≥30 / 博士≥50 | 禁止进入 Phase 2 |
| 索引卡完整性 | 所有必填字段有实质内容 | REVISE 该索引卡 |
| cite_key 唯一性 | 全局无重复 | 修复后重新生成汇总表 |
| 检索策略审查 | Gate 1a.1 = PASS | 修改后重新提交 |
