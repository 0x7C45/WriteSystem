# 科研非数据图表类型目录

> 本目录收录 6 大类共 33 种科研中常用的非数据解释型图表，供图表类型选择与提示词构造参考。
> 每种图表标注了：名称（中/英）、用途、关键元素、推荐工具、典型研究场景。

---

## A. 流程与步骤类（Process & Step Diagrams）

### A1. 流程框图 / Flowchart

- **用途**：展示一系列步骤的执行顺序和分支判断。
- **关键元素**：起止节点、处理步骤、判断菱形、箭头连线、分支标签。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：数据处理管道说明、实验操作规程展示。

### A2. 实验流程图 / Experimental Procedure Diagram

- **用途**：描述一项实验从准备到收尾的完整操作顺序。
- **关键元素**：实验阶段分组（准备/执行/分析）、材料节点、操作步骤、时间标注、条件参数。
- **推荐工具**：Mermaid（`flowchart` + 子图 `subgraph`）
- **典型场景**：论文 Methods 段落的配套流程图，展示样本采集→处理→检测→分析全链路。

### A3. 方法流程图 / Methodology Workflow

- **用途**：说明研究方法的技术路径和决策节点。
- **关键元素**：方法选择节点、技术步骤、中间产物、决策分支、输出结果。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：论文方法论章节，展示"数据采集→预处理→建模→验证"的方法链条。

### A4. 技术路线图 / Technology Roadmap

- **用途**：展示技术发展的阶段目标和里程碑。
- **关键元素**：时间轴、阶段分隔、里程碑节点、关键交付物、依赖连线。
- **推荐工具**：Mermaid（`timeline` 或 `gantt`）
- **典型场景**：项目申报书中的技术路线、论文中研究方案的阶段规划。

### A5. 研究路径图 / Research Path Diagram

- **用途**：展示从问题提出到结论得出的研究推进路径。
- **关键元素**：研究问题、假设、验证手段、中间发现、结论节点、反馈箭头。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：学位论文绪论中的研究路径图，展示"问题→假设→实验→分析→结论"。

### A6. 泳道图 / Swimlane Diagram

- **用途**：在流程中区分不同角色或部门各自的职责和交互。
- **关键元素**：泳道（横向或纵向分区）、角色标签、流程步骤、跨泳道箭头、交接点。
- **推荐工具**：Mermaid（`flowchart` + `subgraph` 分区）
- **典型场景**：多学科协作研究流程、临床研究中医生-患者-实验室的交互流程。

## B. 框架与结构类（Framework & Structure Diagrams）

### B1. 框架图 / Conceptual Framework

- **用途**：展示理论或方案的整体组成和逻辑结构。
- **关键元素**：核心概念模块、层级关系、连接线、外部因素标注。
- **推荐工具**：Mermaid（`block-beta` 或 `flowchart`）
- **典型场景**：论文中"本文研究框架"的总体展示，说明输入→过程→输出的理论结构。

### B2. 系统架构图 / System Architecture Diagram

- **用途**：描述系统的整体组成、各子系统的关系和数据流向。
- **关键元素**：子系统模块、数据流箭头、接口标注、外部系统、系统边界。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：信息系统论文中的系统架构、软件工程项目的架构设计图。

### B3. 模型架构图 / Model Architecture Diagram

- **用途**：展示算法模型或计算模型的内部结构。
- **关键元素**：模型层/组件、输入输出标注、数据维度、参数传递箭头、激活函数。
- **推荐工具**：Graphviz（`dot`）或 Mermaid（`block-beta`）
- **典型场景**：深度学习论文的模型结构图、统计模型的变量关系图。

### B4. 模块组成图 / Module Composition Diagram

- **用途**：展示一个系统或平台的功能模块划分和组成关系。
- **关键元素**：模块节点、包含关系（父子）、模块间依赖箭头、功能标签。
- **推荐工具**：Mermaid（`block-beta`）
- **典型场景**：软件系统设计文档中的模块组成、平台功能架构展示。

### B5. 分层结构图 / Layered Structure Diagram

- **用途**：按照层级展示系统或概念的分层组织。
- **关键元素**：层级标签、层间关系箭头、同层并列项、层边界线。
- **推荐工具**：Mermaid（`block-beta`）
- **典型场景**：网络协议栈分层图、企业架构分层图（战略层/业务层/技术层/数据层）。

### B6. 平台结构图 / Platform Structure Diagram

- **用途**：展示平台型产品的整体结构和各子系统关系。
- **关键元素**：核心平台层、应用层、基础设施层、用户入口、API 接口、外部集成。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：SaaS 平台架构图、数据分析平台的整体结构展示。

## C. 概念与定义类（Concept & Definition Diagrams）

### C1. 概念图 / Concept Map

- **用途**：展示核心概念及其子概念、属性和关系的网络结构。
- **关键元素**：概念节点、关系连线、连接词标签、层级结构、交叉连接。
- **推荐工具**：Mermaid（`mindmap` 或 `flowchart`）
- **典型场景**：文献综述中梳理核心概念的内涵与外延、教学材料中的概念讲解。

### C2. 概念拆解图 / Concept Breakdown Diagram

- **用途**：将一个复杂概念逐层拆解为可操作的子概念或维度。
- **关键元素**：顶层概念、拆解层级、子维度节点、拆解逻辑标签（"包含"/"分为"/"体现为"）。
- **推荐工具**：Mermaid（`mindmap`）
- **典型场景**：论文中对"数字化素养""智慧校园"等复合概念的维度拆解。

### C3. 分类图 / Classification Diagram / Taxonomy

- **用途**：按照分类标准展示类别之间的层级或并列关系。
- **关键元素**：分类根节点、分类标准标签、类别节点、层级连线、示例标注。
- **推荐工具**：Mermaid（`mindmap` 或 `flowchart`）
- **典型场景**：文献综述中的研究方法分类、领域术语分类体系。

### C4. 术语关系图 / Terminology Relationship Diagram

- **用途**：展示一组术语之间的定义关系（同义、包含、对立、依赖等）。
- **关键元素**：术语节点、关系类型标签（同义/上下位/对立/相关）、关系箭头。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：跨学科研究中统一术语体系、论文中对易混概念的辨析。

### C5. 本体图 / Ontology Diagram

- **用途**：按照本体工程规范展示概念的类、属性、实例和形式化关系。
- **关键元素**：类节点、实例节点、对象属性连线、数据属性标注、继承箭头（is-a）、部分整体关系（part-of）。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：知识图谱构建论文中的本体设计、信息系统领域建模。

### C6. 知识图谱式关系图 / Knowledge Graph Style Diagram

- **用途**：以知识图谱的视觉风格展示实体之间的多类型关系网络。
- **关键元素**：实体节点（可按类型着色）、关系边（带标签）、多跳路径、图例。
- **推荐工具**：Graphviz（`dot`，使用 `neato` 或 `sfdp` 布局）
- **典型场景**：领域知识体系展示、论文中核心实体关系的全局概览。

## D. 机制与因果类（Mechanism & Causal Diagrams）

### D1. 作用机制图 / Mechanism of Action Diagram

- **用途**：展示某个现象或过程的内在作用机制。
- **关键元素**：触发因素、中间过程节点、作用路径、结果节点、反馈回路、抑制/促进标注。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：生物学论文中的信号通路机制、经济学中的政策传导机制。

### D2. 因果路径图 / Causal Path Diagram

- **用途**：展示变量之间的因果关系链条。
- **关键元素**：因果变量节点、因果箭头（方向性）、中介变量、调节变量、混淆变量标注。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：结构方程模型（SEM）的路径图、流行病学因果推断的路径展示。

### D3. 理论模型图 / Theoretical Model Diagram

- **用途**：展示理论中各要素之间的逻辑关系和假设。
- **关键元素**：理论构念节点、假设路径箭头、假设编号标注、控制变量、解释标注。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：管理学/教育学论文中的理论模型图、假设关系图。

### D4. 变量关系图 / Variable Relationship Diagram

- **用途**：展示研究中自变量、因变量、中介变量和调节变量之间的关系。
- **关键元素**：变量节点、关系箭头、关系类型标签（正相关/负相关/调节/中介）、显著性标注。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：实证研究论文的变量关系图、回归分析中的变量体系图。

### D5. 影响机制图 / Influence Mechanism Diagram

- **用途**：展示某个因素如何通过多条路径影响结果。
- **关键元素**：影响因素、多条影响路径、中介节点、直接/间接效应标注、边界条件。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：政策评估中的影响机制分析、教育学中的教学效果影响路径。

### D6. 闭环反馈图 / Feedback Loop Diagram

- **用途**：展示系统中正反馈或负反馈的循环机制。
- **关键元素**：反馈环节节点、循环箭头、正/负反馈标注、延迟标记、系统边界。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：系统动力学中的因果回路图、生态系统的反馈机制、组织学习的闭环模型。

## E. 场景与交互类（Scenario & Interaction Diagrams）

### E1. 应用场景图 / Application Scenario Diagram

- **用途**：展示系统或方法在不同场景下的应用方式和差异。
- **关键元素**：场景分区、场景名称、参与主体、场景内流程、跨场景连接。
- **推荐工具**：Mermaid（`flowchart` + `subgraph`）
- **典型场景**：论文中"系统应用场景"的展示、产品需求文档中的使用场景。

### E2. 用户旅程图 / User Journey Map

- **用途**：展示用户在完成目标过程中的关键触点和体验。
- **关键元素**：用户角色、阶段划分、触点节点、行为/情绪/痛点标注、机会点。
- **推荐工具**：Mermaid（`flowchart`）
- **典型场景**：用户研究论文中的用户体验旅程、产品设计中的人物旅程图。

### E3. 多主体协作图 / Multi-Agent Collaboration Diagram

- **用途**：展示多个参与主体之间的协作关系和交互方式。
- **关键元素**：主体节点（角色/组织/系统）、协作连线、交互内容标签、时序标注。
- **推荐工具**：Mermaid（`sequenceDiagram` 或 `flowchart`）
- **典型场景**：跨组织合作研究流程、临床多中心试验的协作网络。

### E4. 交互流程图 / Interaction Flow Diagram

- **用途**：展示系统与用户之间的交互步骤和信息流转。
- **关键元素**：用户操作节点、系统响应节点、信息流向箭头、界面状态标注。
- **推荐工具**：Mermaid（`sequenceDiagram`）
- **典型场景**：人机交互论文的交互流程、系统需求文档的用户操作流程。

### E5. 系统使用情境图 / System Usage Context Diagram

- **用途**：展示系统在真实使用环境中的上下文和边界。
- **关键元素**：系统核心、用户角色、外部系统、环境条件、交互接口、系统边界框。
- **推荐工具**：Graphviz（`dot`）
- **典型场景**：论文中系统使用情境的总览图、需求分析中的上下文图。

### E6. 时序图 / Sequence Diagram

- **用途**：展示多个参与者之间的消息传递顺序和交互时序。
- **关键元素**：参与者（participant）、生命线、消息箭头（同步/异步/返回）、激活条、条件分支（alt/else）、循环（loop）、注释（Note）。
- **推荐工具**：Mermaid（`sequenceDiagram`）
- **典型场景**：系统间通信协议流程（如 MQTT 握手）、人机交互时序、API 调用链展示。

## F. 总结与传播类（Summary & Communication Diagrams）

### F1. 图形摘要 / Graphical Abstract

- **用途**：用一张图概括论文核心内容，供期刊目录和快速浏览使用。
- **关键元素**：研究问题、方法/模型、关键发现、核心结论、视觉逻辑主线（左→右 或 上→下）。
- **推荐工具**：Graphviz（`dot`）或 Mermaid（`flowchart`）
- **典型场景**：期刊投稿要求的 Graphical Abstract、学术海报的核心图。

### F2. 视觉摘要 / Visual Summary

- **用途**：将文字内容浓缩为视觉化摘要，便于快速理解。
- **关键元素**：关键信息节点、视觉分组、连接逻辑、图标辅助、文字精简。
- **推荐工具**：Mermaid（`mindmap` 或 `flowchart`）
- **典型场景**：论文答辩PPT的核心内容摘要、学术报告的可视化总结。

### F3. 论文贡献图 / Contribution Diagram

- **用途**：展示论文的主要贡献点和创新之处。
- **关键元素**：贡献点节点（按类型分组）、贡献之间的关系、与现有工作的对比标注。
- **推荐工具**：Mermaid（`flowchart` 或 `mindmap`）
- **典型场景**：论文 Conclusion 或 Introduction 中的贡献总结图。

### F4. 研究亮点图 / Research Highlights Diagram

- **用途**：以要点形式展示研究的核心亮点和关键发现。
- **关键元素**：亮点条目、编号/排序、图标辅助、分类标签（方法/发现/应用）。
- **推荐工具**：Mermaid（`mindmap`）
- **典型场景**：论文投稿时的 Research Highlights、项目汇报的成果亮点。

### F5. 对比总结图 / Comparative Summary Diagram

- **用途**：对比展示多个方案、方法或研究的异同。
- **关键元素**：对比维度、对比对象列、差异标注、优势/劣势标注、汇总结论。
- **推荐工具**：Mermaid（`flowchart`）或 Graphviz（`dot`）
- **典型场景**：文献综述中的方法对比图、Related Work 段落的方案比较。
