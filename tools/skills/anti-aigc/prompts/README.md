## 使用方法

### 1. 自动调用（推荐）

在 `anti_aigc_pipeline.py` 的 Phase 2 中自动根据段落类型选择对应模板：

```python
def _build_adversarial_prompt(self, para: Dict) -> str:
    """根据段落类型选择Prompt模板"""
    para_type = para["type"]  # "摘要", "引言", "方法" 等
    
    template_map = {
        "摘要": "摘要_对抗Prompt.md",
        "引言背景": "引言_对抗Prompt.md",
        "文献综述": "文献综述_对抗Prompt.md",
        "方法": "方法_对抗Prompt.md",
        "结果分析": "结果分析_对抗Prompt.md",
        "讨论": "讨论_对抗Prompt.md",
        "结论": "结论_对抗Prompt.md"
    }
    
    template_file = template_map.get(para_type, "摘要_对抗Prompt.md")
    template = self._load_prompt_template(template_file)
    
    return template.replace("{原始段落}", para["text"])
```

### 2. 手动使用

复制对应模板的内容，将 `{原始段落}` 替换为实际文本，发送给LLM API。

## Prompt 设计原则

### 1. 核心目标
- **消除AI痕迹**：打破模板句式、高频虚词、机械结构
- **保持学术规范**：论点清晰、逻辑严谨、表达准确
- **注入人类不完美**：限定词、不确定性、适度谨慎

### 2. 五层改写要求

每个Prompt模板包含5个层次的改写要求：

1. **句式多样性** - 打破模板句式，长短句交错
2. **词汇自然化** - 避免AI高频词，使用同义词变体
3. **结构/逻辑自然化** - 打破机械结构，允许非线性
4. **学术不完美** - 限定词、不确定性、承认局限
5. **完整性保障** - 引用、数据、术语必须保留

### 3. 解码参数策略

| 参数 | 推荐范围 | 作用 |
|-----|---------|-----|
| temperature | 0.95-1.15 | 控制多样性，偏高以增加随机性 |
| top_p | 0.80-0.90 | 核采样，保持合理性的同时允许变化 |
| frequency_penalty | 0.35-0.50 | 降低词汇重复 |
| presence_penalty | 0.20-0.30 | 鼓励使用多样的表达 |

**注意**：不同段落类型的参数有所差异：
- 摘要、讨论：温度稍高（1.1+），鼓励更多变化
- 结果分析：温度较低（0.95），保持数据准确性
- 方法：温度中等（1.0），平衡清晰性和自然性

## 示例效果

### 改写前（AI痕迹明显）
> 随着人工智能技术的快速发展，智能教育系统在个性化学习中发挥着越来越重要的作用。首先，本研究探讨了基于深度学习的学习路径推荐算法。然后，通过实验验证了该算法的有效性。最后，结果表明该系统能够显著提高学生的学习成绩。

**AI痕迹**：
- ❌ "随着...的发展" - AI高频开头
- ❌ "首先...然后...最后..." - 模板化连接词
- ❌ "发挥着越来越重要的作用" - 空洞表达
- ❌ "显著提高" - 过度自信，缺乏限定

### 改写后（自然学术风格）
> 智能教育系统近年来在个性化学习领域展现出巨大潜力。本研究围绕深度学习驱动的学习路径推荐算法展开探讨，实验数据显示该算法在学习效率提升方面具有一定效果。初步结果表明，系统对学生成绩的改善作用相对明显，但仍需进一步验证其在不同学习场景下的适用性。

**改进点**：
- ✅ 打破模板开头，直接切入主题
- ✅ 去除"首先-然后-最后"序列
- ✅ 使用"具有一定效果"、"相对明显"等限定词
- ✅ 加入"仍需进一步验证"的谨慎表达

## 注意事项

### ⚠️ 必须遵守

1. **引用完整性**：所有 `[cite_key]` 必须保留，这是硬性要求
2. **数据准确性**：数值、百分比、统计结果不得修改
3. **术语一致性**：关键术语、模型名称保持一致

### ⚠️ 避免过度

1. 不要为了"自然"而牺牲学术规范
2. 不要注入过多不确定性导致观点模糊
3. 不要过度打散结构导致逻辑混乱
