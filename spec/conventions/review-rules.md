# 审查规则细则

> 版本: v1.0.0 | 所属: WriteSystem spec/conventions
> 关联: pipeline-spec.md（触发节点）、quality-and-tools.md（判定阈值）

---

## 约定

- 每条审查项标记为 **ERROR**（阻断，必须修复）或 **WARNING**（不阻断，记录并提示）
- 任一 ERROR 存在 → 该 Gate 判定为 REVISE
- 仅有 WARNING → 该 Gate 判定为 PASS（附警告清单）
- ERROR 和 WARNING 同时存在 → REVISE
- 审查方只输出结构化的检查结果，不直接修改文件

---

# 第一部分：content-reviewer 规则

content-reviewer 负责**逻辑、结构、范围和学术严谨性**的只读审查。

---

## C1 — 订单摘要完整性

```
阶段: Phase 0, Gate 0.1
审查对象: 00_订单信息/订单摘要.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| C1.1 | 「基本信息」表格中所有字段均已填写，不得有「待填写」 | ERROR |
| C1.2 | 论文类型明确（毕业论文/期刊/课程） | ERROR |
| C1.3 | 学科领域填写到二级学科或具体方向 | ERROR |
| C1.4 | 字数要求为具体数字，非范围性描述（如"15000字"而非"1-2万字"） | ERROR |
| C1.5 | 截止日期为有效的未来日期 | ERROR |
| C1.6 | 引用格式标准明确（如 GB/T 7714-2015） | ERROR |
| C1.7 | 「特殊要求」字段至少有一条具体内容 | WARNING |

---

## C2 — 格式规范完整性

```
阶段: Phase 0, Gate 0.2
审查对象: 00_订单信息/格式规范.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| C2.1 | 「页面设置」表的纸张大小、四边边距、行距均已填写 | ERROR |
| C2.2 | 「字体规范」表至少包含正文、一级标题、二级标题 | ERROR |
| C2.3 | 正文的字体和字号与模板一致（与 `analyze_docx_template` 输出对照） | ERROR |
| C2.4 | 引用格式标准与订单摘要中声明的一致 | ERROR |
| C2.5 | 图表编号方式有明确规则（按章/全篇连续） | ERROR |
| C2.6 | 所有值来自模板分析结果，非人工猜测（与 `analyze_docx_template` 输出逐项对照） | WARNING |

## C3 — 检索策略质量

```
阶段: Phase 1a, Gate 1a.1
审查对象: 04_参考文献/literature_cards/_检索策略.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| C3.1 | 中文检索词 ≥ 5 个，英文检索词 ≥ 5 个 | ERROR |
| C3.2 | 检索源 ≥ 3 个（覆盖中文和英文） | ERROR |
| C3.3 | 每类检索词覆盖了研究主题的核心概念和同义词 | ERROR |
| C3.4 | 检索时间范围明确声明 | WARNING |
| C3.5 | 检索策略包含排除标准（如排除非同行评议、排除会议摘要） | WARNING |

---

## C4 — 索引卡质量

```
阶段: Phase 1a, Gate 1a.2
审查对象: 04_参考文献/literature_cards/{作者}_{年份}.md (每张)
```

| # | 检查项 | 级别 |
|---|--------|------|
| C4.1 | 「核心发现」字段 ≥ 150 字，包含研究问题、方法、发现、结论四个要素 | ERROR |
| C4.2 | 「研究方法」字段 ≥ 80 字，包含实验设计或数据来源 | ERROR |
| C4.3 | DOI/URL 字段不为空，且链接格式为完整 URL | ERROR |
| C4.4 | 「与本文关系」字段有具体章节引用（如「第2章」），非泛泛而谈 | ERROR |
| C4.5 | 索引卡总数 ≥ 该论文类型的最低要求（quality-and-tools.md 引用标准） | ERROR |
| C4.6 | 「关键摘录」字段至少 1 条直接引用 | WARNING |

审查对象: 03_计划与方案/开题报告.md + 文献索引卡汇总
```

| # | 检查项 | 级别 |
|---|--------|------|

| C5.2 | 研究缺口分析有实质内容（非「该领域研究较少」这类空话） | ERROR |
| C5.3 | 方法选择有至少 2 篇文献作为支撑 | ERROR |
| C5.4 | 提出了与前人工作的明确对比点 | WARNING |
| C5.5 | 近 5 年文献引用占比 ≥ 50%（quality-and-tools.md 引用标准） | WARNING |

## C6 — 学术严谨性（计划层）

```
阶段: Phase 1b, Gate 1b.G2
审查对象: 03_计划与方案/开题报告.md
审查定位: 「这个研究计划在学术上站得住吗？」 — 把关方向
```

| # | 检查项 | 级别 |
|---|--------|------|
| C6.1 | 研究问题表述为可验证/可回答的命题（非「探讨...」这类模糊动词） | ERROR |
| C6.2 | 研究问题有明确的边界（什么在研究范围内、什么不在） | ERROR |
| C6.3 | 声称的研究方法在文献中有先例（至少 1 篇索引卡支持） | ERROR |
| C6.4 | 预期贡献不夸大（如「填补空白」「首次提出」需有充分证据） | ERROR |
| C6.5 | 未声称超出数据范围或当前能力的结论 | ERROR |
| C6.6 | 术语使用规范且在全文有一致定义 | WARNING |
| C6.7 | 研究假设（如有）明确列出 | WARNING |

## C7 — 结构连贯性

```
阶段: Phase 1b, Gate 1b.G4
审查对象: 03_计划与方案/骨架大纲.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| C7.1 | 章标题之间有逻辑递进关系（前章为后章铺垫，非独立堆放） | ERROR |
| C7.2 | 每章节数 ≥ 2（不存在只有一节的章） | ERROR |
| C7.3 | 各章节的段落要点有因果关系或论证推进 | ERROR |
| C7.4 | 全文从背景→问题→方法→分析→结论有一条清晰主线 | ERROR |
| C7.5 | 无重复内容（同一论点不计划出现在两个章节中） | WARNING |
| C7.6 | 章标题不包含动词（如「设计了一个...」→ 应为「...的设计」） | WARNING |

## C8 — 章节计划审查

```
阶段: Phase 2, Gate 2.1
审查对象: 即将撰写的章节计划（章节文件头部注释或独立计划）
```

| # | 检查项 | 级别 |
|---|--------|------|
| C8.1 | 本章与前一章有明确的衔接句/衔接段计划 | ERROR |
| C8.2 | 本章列出的引用索引卡在 04_参考文献/ 中均存在 | ERROR |
| C8.3 | 本章计划覆盖骨架大纲中该章的所有段落要点 | ERROR |
| C8.4 | 本章所需数据在 data_facts.md 中有来源 | WARNING |
| C8.5 | 本章与后一章的过渡有预备 | WARNING |

## C9 — 章节质量（执行层）

```
阶段: Phase 2, Gate 2.2
审查对象: 05_撰写过程/第N章_{标题}.md
审查定位: 「这一章写得到位吗？」 — 把关落地质量，而非逐项指标
```

| # | 检查项 | 级别 |
|---|--------|------|
| C9.1 | 本章开篇有引导段（说明本章要解决的问题和结构安排） | ERROR |
| C9.2 | 每个节回答了骨架大纲中该节提出的问题 | ERROR |
| C9.3 | 段落之间有逻辑连接（过渡词、因果链、递进关系），阅读顺畅 | ERROR |
| C9.4 | 本章结尾有小结段（概括本章结论，预告下一章） | ERROR |
| C9.5 | 论证充分性: 每个论点有足够的论据支撑（数据/引用/推理），不出现「显然」「众所周知」作为论据 | ERROR |
| C9.6 | 无逻辑跳跃: 从 A 到 D 不跳过 B 和 C | WARNING |
| C9.7 | 段落长度均衡: 无不正常的长段（>500 字无分段）或过短段（连续3段 <50 字） | WARNING |
| C9.8 | 无冗余重复: 同一论点不在本章内以不同措辞重复出现 | WARNING |

# 第二部分：data-auditor 规则

data-auditor 负责**数据一致性、完备性和捏造检测**的只读审查。

---

## D1 — 骨架数据完备性

```
阶段: Phase 1b, Gate 1b.G3
审查对象: 03_计划与方案/骨架大纲.md + 02_工作素材/data_facts.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| D1.1 | 骨架中所有声称的数据点在 data_facts.md 中有对应条目 | ERROR |
| D1.2 | 骨架中未声称 data_facts.md 中不存在的数据 | ERROR |
| D1.3 | 骨架中引用的数值范围在 data_facts.md 的记录范围内 | ERROR |
| D1.4 | 如有多个数据集，骨架中明确标注了数据来源 | WARNING |

---

## D2 — 章节数据准确性

```
阶段: Phase 2, Gate 2.2
审查对象: 05_撰写过程/第N章_{标题}.md + data_facts.md
```

| # | 检查项 | 级别 |
|---|--------|------|
| D2.1 | 章节中出现的每个数值可追溯到 data_facts.md 的具体条目 | ERROR |
| D2.2 | 数值与 data_facts.md 中的记录一致（精确匹配） | ERROR |
| D2.3 | 单位使用正确且全文统一 | ERROR |
| D2.4 | 统计推断的结论与数据中的统计量一致（如声称「显著」时有 p<0.05） | ERROR |
| D2.5 | 小数点精度在全文中保持一致（如同一变量不应一处保留 2 位、另一处保留 4 位） | WARNING |
| D2.6 | 无凭空捏造的数据（找不到来源且不能从已有数据推导） | ERROR |

## D3 — 全文数据一致性

```
阶段: Phase 2, Gate 2.3

| # | 检查项 | 级别 |

| D3.5 | 样本量在全文所有分析中一致（总 N 不变，除非声明了子集分析） | WARNING |

# 第三部分：advisor 里程碑判定规则

advisor 是**唯一有权限做 PASS/REVISE/DISCUSS 判定的角色**。

---

## 判定体系

```
PASS      — 通过，进入下一阶段
REVISE    — 有问题，但可修复。返回指定步骤修改后再提交
DISCUSS   — 有根本性问题，Agent 无法自行决策。暂停管线，等待用户
```

---

## A1 — M1 方向确认

```
触发: Phase 1a + 1b 完成
审查对象: 订单摘要 + 索引卡汇总 + 开题报告 + content-reviewer 的 C1-C7 结果
```

| # | 检查项 | 判定规则 |
|---|--------|---------|
| A1.1 | 选题可行性 | 研究问题在现有文献和资源条件下可完成 → PASS；问题过大/过小 → REVISE；方向本身有争议 → DISCUSS |
| A1.2 | 文献基础 | 索引卡数量达标 + 文献覆盖核心方向 + 有研究缺口 → PASS；数量不足 → REVISE；文献明显偏颇 → DISCUSS |
| A1.3 | 创新性/贡献 | 有明确区别于前人的贡献点 → PASS；与前人高度重叠 → DISCUSS |
| A1.4 | content-reviewer 的 ERROR | 全部清零 → 参与判定；仍有 ERROR → 强制 REVISE |

**综合判定**：A1.1-A1.4 全部 PASS → 综合 PASS；任一 DISCUSS → 综合 DISCUSS；其他 → REVISE。

---

## A2 — M2 骨架确认

```
触发: Phase 1b 完成
审查对象: 骨架大纲 + 核心论点摘要 + content-reviewer C6-C7 + data-auditor D1
```

| # | 检查项 | 判定规则 |
|---|--------|---------|
| A2.1 | 研究问题 | 清晰、可验证、有边界 → PASS；模糊 → REVISE |
| A2.2 | 方法论 | 方法恰当、有文献支撑、可操作 → PASS；方法有缺陷 → REVISE；方法不可行 → DISCUSS |
| A2.3 | 框架逻辑 | 章-节-段有清晰逻辑链 → PASS；逻辑跳跃或重复 → REVISE |
| A2.4 | 复杂度 | 工作量与论文学位/类型匹配 → PASS；过重/过轻 → REVISE |
| A2.5 | 审查结果 | C6/C7/D1 的全部 ERROR 已清零 |

**综合判定**：同 M1 逻辑。

---

## A3 — M3 草稿审查

```
触发: Phase 2 完成
审查对象: 正文草稿.md + 引用注册表 + C9/D2/D3 结果
```

| # | 检查项 | 判定规则 |
|---|--------|---------|
| A3.1 | 全文连贯性 | 章节间过渡自然、论点层层推进 → PASS；有明显断点 → REVISE |
| A3.2 | 学术规范 | 引用格式统一、无抄袭嫌疑、术语一致 → PASS；有问题 → REVISE |
| A3.3 | 引用一致性 | validate_citations 通过、引用注册表无冲突 → PASS |
| A3.4 | 审查结果 | C9/D2/D3 的全部 ERROR 已清零 |

**综合判定**：同 M1 逻辑。M3 的 REVISE 最多允许 2 轮迭代。

---

## A4 — M4 交付确认

```
触发: Phase 3 (+ 3.5) 完成
审查对象: 终稿.docx + 终审报告 + C10/D4 + AIGC 评估
```

| # | 检查项 | 判定规则 |
|---|--------|---------|
| A4.1 | 排版质量 | DOCX 校验全部通过 → PASS |
| A4.2 | AIGC 风险 | 综合分 < 中风险阈值 → PASS；否则 → REVISE（触发 Phase 3.5） |
| A4.3 | 最终完整性 | 所有交付物齐全、终审报告无 ERROR → PASS |
| A4.4 | 审查结果 | C10/D4 的全部 ERROR 已清零 |

**综合判定**：A4.1-A4.4 全部 PASS → 综合 PASS → 项目进入 REVIEW 状态，等待用户确认交付。

---

## REVISE 后的回退路径

| 里程碑 | REVISE 触发时 | 回退到 |
|--------|-------------|--------|
| M1 | 选题/文献/创新性问题 | Phase 1a (文献) 或 Phase 1b (开题)，取决于哪个维度失败 |
| M2 | 方法/框架/复杂度问题 | Phase 1b 阶段 B 或 C，取决于哪个维度失败 |
| M3 | 连贯性/规范/引用问题 | Phase 2 对应章节的 Gate 2.2 |
| M4 | 排版/AIGC/完整性问题 | Phase 3 对应步骤 |

## DISCUSS 的处理

DISCUSS 出现时，advisor 必须：
1. 写入 `导师确认态.md`，注明 DISCUSS 的具体维度和理由
2. 将文件末尾状态改为 `BLOCKED`
3. 向用户提出明确的待决策问题（选择题形式，≤3 个选项）
4. 不自动执行任何操作，等待用户回复
