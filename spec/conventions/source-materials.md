# 原始资料与工作素材管理规范

> 版本: v1.0.0 | 所属: WriteSystem spec/conventions

---

## 一、核心原则：用户只投放，AI 负责分类

```
用户动作:  把所有资料扔进 原始资料/（不管类型，不管格式）
AI 动作:   扫描 → 识别类型 → 分类 → 登记 → 自动归位
```

用户不需要知道 `requirements/ references/ attachments/` 的区别。
AI（paper-order-analyst, Phase 0）负责这件事。

---

## 二、投放点：`00_订单信息/原始资料/`

### 目录结构

```
00_订单信息/原始资料/        ← 用户唯一投放点
├── 任务书.pdf              ← 用户扔的
├── 格式模板.docx           ← 用户扔的（AI 识别后复制到 01_格式模板/）
├── 师兄论文终稿.docx       ← 用户扔的
├── 期刊范文.pdf            ← 用户扔的
├── 导师批注_v2.docx        ← 用户扔的
├── 自己写的思路脑图.png    ← 用户扔的
├── 自己写的笔记.md         ← 用户扔的
├── ...（任何格式，任何数量）
└── README.md               ← AI 自动生成/更新（分类清单）
```

### 规则

- **不分子目录** — 用户把所有文件平铺扔进去
- **不限格式** — PDF / DOCX / JPG / PNG / MD / TXT / CSV / XLSX 全部接受
- **不限数量** — 1 个也行，50 个也行
- **用户不需要命名规范** — 原始文件名保留不动
- **README.md 由 AI 生成** — 用户不需要手动登记

---

## 三、AI 自动分类流程 (Phase 0 Step 0.1)

paper-order-analyst 执行以下步骤：

```
Step 0.1a  扫描原始资料/
  → 列举所有文件（含文件大小、格式）
  → 对于 .docx .pdf .md，读取内容摘要

Step 0.1b  识别每个文件的类型
  识别规则（按优先级）:

  ① 格式模板类 → 校内官方模板文件
     特征: 文件名含「模板」「格式」「规范」，内容含样式定义表
     → 复制到 01_格式模板/
     → 原始文件保留在 原始资料/ 不动

  ② 需求类 (requirements) → 任务书、要求、批注
     特征: 文件名含「任务书」「要求」「评分」「批注」「通知」
     或内容为对论文的明确要求

  ③ 参考类 (references) → 范文、师兄论文
     特征: 完整的论文文档，内容含摘要、目录、章节

  ④ 附件类 (attachments) → 学校提供的流程文件
     特征: 文件名含「开题报告」「中期检查」「答辩」「申请」

  ⑤ 数据类 → 问卷、实验、测量数据
     特征: .csv .xlsx 或明显的表格数据
     → 复制到 02_工作素材/raw/

  ⑥ 图片类 → 无文字内容的纯图片
     特征: .jpg .png，无结构化的文档内容
     → 复制到 02_工作素材/photos/

  ⑦ 用户笔记 → 用户自己的文字总结
     特征: .md .txt，内容为用户的思路整理

  ⑧ 未识别 → 标记为 UNKNOWN，保留在原始资料/

Step 0.1c  生成分类清单
  → 写入 原始资料/README.md
  → 格式见 file-schemas.md
```

```markdown

|--------|------|---------|

## 附件

| 文件名 | 说明 |
|--------|------|

## 用户笔记

| 文件名 | 摘要 |
|--------|------|

|--------|------|

```

## 五、`01_格式模板/`

### 存储规则

```
01_格式模板/
├── {学校名}_格式模板.docx       ← AI 从 原始资料/ 识别并复制过来
└── {学校名}_格式模板分析.md      ← AI 用 analyze_docx_template 生成
```

- 格式模板由 AI 从 `原始资料/` 中**自动识别并复制**，用户不需要手动操作
- 一个项目只放一套模板

## 六、`02_工作素材/`

### 目录结构

```
02_工作素材/
├── data_facts.md            ← 必写：实验数据摘要（AI 辅助填写）
├── scripts/                 ← 项目内临时脚本
├── raw/                     ← 原始数据（AI 从 原始资料/ 复制来的 .csv .xlsx）
├── charts/                  ← 生成的图表
└── photos/                  ← 照片/截图（AI 从 原始资料/ 复制来的 .jpg .png）
```

### data_facts.md

格式见 `file-schemas.md`。AI 在 Phase 0 自动生成模板，用户在 Phase 1-2 填写。

---

## 七、禁止项

| 禁止 | 原因 |
|------|------|
| 用户手动分子目录 | 分类是 AI 的工作 |
| 用户自己写 README.md | README.md 由 AI 自动生成和更新 |
| 格式模板只放 原始资料/ 不复制到 01/ | AI 必须复制过去 |
| 数据文件只放 原始资料/ 不复制到 02/ | AI 必须复制过去 |
