# 文件命名规范

## 项目目录

```
格式: {类型前缀}{序号}_{英文slug}
示例: writing1, thesis1, report1

类型前缀:
  writing   — 论文撰写
  thesis    — 降重/改写
  report    — 研究报告
```

## Phase 目录 (00-07)

```
00_订单信息/     — 需求定义
01_格式模板/     — 目标格式模板
02_工作素材/     — 原始数据/图表/代码
03_计划与方案/   — 开题/大纲/论点
04_参考文献/     — 文献索引卡
05_撰写过程/     — 逐章草稿
06_最终交付/     — 终稿+报告
07_归档快照/     — 项目归档摘要
```

## 章节文件

```
格式: 第{N}章_{标题}.md
示例: 第1章_绪论.md, 第3章_处理器指令集与微架构设计.md

规则:
  - N 从 0 开始（摘要/封面用 0a/0b/0c）
  - 标题与骨架大纲一致
  - 不用空格，用下划线
```

## 文献索引卡

```
格式: {作者}_{年份}.md
示例: Patterson_2017.md, Bao_2021.md

规则:
  - 作者用英文姓氏，首字母大写
  - 多年份: Patterson_2017,2023.md
  - 多作者用第一作者: Waterman_2019.md
  - 同名同年加后缀: Liu_2022a.md, Liu_2022b.md
```

## 交付物

```
格式: {论文标题}_{版本标识}.docx

版本标识:
  草稿版  — 初排未审
  格式版  — 已套模板
  终稿    — 最终交付（归档时唯一保留的 docx）
```

## 脚本与工具

```
项目脚本: 02_工作素材/scripts/{功能名}.py
仓库工具: tools/{功能名}.sh
```

## 禁止项

- 项目根目录下禁止放置 .py / .docx / test_* 文件
- 禁止裸放 template.docx — 应放在 01_格式模板/
- 禁止保留已解压的 .zip 源文件
- 禁止在 06_最终交付 中保留多版本文档（只留终稿）
