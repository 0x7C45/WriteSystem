# 文件格式 Schema

> 版本: v1.0.0 | 所属: WriteSystem spec/conventions
> 关联: pipeline-spec.md（使用场景）、source-materials.md（素材规范）

---

## 概述

本规范定义管线中所有**跨 Agent 传递的关键文件**的准确格式。
每个 Schema 包含：哪个 Phase 产出、谁写入、谁读取、字段定义、示例。

---

## Schema 1: 订单摘要.md

```
产出: Phase 0 | 写入: order-analyst | 读取: 所有后续 Agent
路径: 00_订单信息/订单摘要.md
```

### Frontmatter (YAML)

```yaml
---
repo_spec_version: v1.0.0
project_version: 1
created: 2026-06-04
status: active
pipeline:
  interaction_level: coarse     # coarse | fine | minimal
  revise_mode: auto             # auto | manual
  max_revise_rounds: 2
---
```

### 正文

```markdown
# 订单摘要

> 项目: {project_name} | 创建: YYYY-MM-DD

## 基本信息

| 字段 | 值 |
|------|-----|
| 论文标题 | {完整中文标题} |
| 英文标题 | {English Title} |
| 类型 | 本科毕业论文 |
| 学科领域 | {一级学科 / 二级学科} |
| 字数要求 | 15000 字（中文） |
| 截止日期 | YYYY-MM-DD |
| 引用格式 | GB/T 7714-2015（顺序编码制） |
| 学校/期刊 | {学校/期刊全称} |

## 章节结构预览

| 章节 | 标题 | 预计字数 | 关键内容 |
|------|------|---------|---------|
| 第1章 | 绪论 | ~2000 | 背景、问题、意义 |
| 第2章 | 文献综述 | ~3000 | ... |

## 特殊要求

1. 必须有至少 10 张图表
2. 英文摘要必须经过润色
3. (其他特殊要求)

## 确认记录

| 日期 | 确认项 | 判定 | 备注 |
|------|--------|------|------|
| | | | |
```

### 规则

- 「基本信息」表格的所有字段必须填写，不得为「待填写」
- 「章节结构预览」在 Phase 1b 骨架确认后补充
- 「确认记录」由 advisor 的 M1-M4 判定自动追加

## Schema 2: 格式规范.md

```
产出: Phase 0 | 写入: order-analyst | 读取: formatter (Phase 3)
路径: 00_订单信息/格式规范.md
```

```markdown
# 格式规范

> 项目: {project_name} | 模板位置: `01_格式模板/`

## 页面设置

| 项目 | 值 |
|------|-----|
| 纸张大小 | A4 |
| 上边距 | 2.54 cm |
| 下边距 | 2.54 cm |
| 左边距 | 3.18 cm |
| 右边距 | 3.18 cm |
| 行距 | 1.5 倍行距 |

## 字体规范

| 元素 | 字体 | 字号 | 加粗 | 对齐 | 段前/段后 |
|------|------|------|------|------|----------|
| 论文标题 | 黑体 | 小二 | 是 | 居中 | — |
| 一级标题 | 黑体 | 三号 | 是 | 左对齐 | 12pt/6pt |
| 二级标题 | 黑体 | 四号 | 是 | 左对齐 | 6pt/3pt |
| 三级标题 | 黑体 | 小四 | 是 | 左对齐 | 3pt/0pt |
| 正文 | 宋体 | 小四 | 否 | 两端对齐 | 0pt/0pt |
| 摘要正文 | 宋体 | 五号 | 否 | 两端对齐 | — |
| 参考文献 | 宋体 | 五号 | 否 | 左对齐 | — |
| 页眉 | 宋体 | 五号 | 否 | 居中 | — |
| 页码 | Times New Roman | 五号 | 否 | 居中 | — |

## 章节编号

- 一级标题: 1, 2, 3, ...
- 二级标题: 1.1, 1.2, ...
- 三级标题: 1.1.1, 1.1.2, ...

## 引用格式

- 标准: GB/T 7714-2015
- 引用方式: 顺序编码制（[1], [2], [3]）
- 参考文献排列: 按正文引用顺序

## 图表规范

- 图标题: 图下方居中，「图1-1 {标题}」(按章编号)
- 表标题: 表上方居中，「表1-1 {标题}」(按章编号)
- 图表内文字: 不小于五号
- 分辨率: ≥ 300 DPI（打印用）

## 特殊页面

- 封面: 使用模板中的封面页
- 声明页: 使用模板中的声明页
- 目录: 自动生成（含二级标题）
- 摘要: 中文在前，英文在后，各占一页
```

### 规则

- 所有字段必须从 `analyze_docx_template` 的输出中提取，不能凭感觉填
- 「字体规范」表至少包含正文和 1-3 级标题
- 如果模板不包含某元素（如三级标题），标注为「不适用」
- ★ **优先级**: 高校模板提取值 > 本规范 Schema 示例值。当高校模板的要求与 Schema 示例不一致时，以高校模板为准

- 文件末尾的「状态」是单行：`ACTIVE` | `READY_FOR_DELIVERY` | `BLOCKED`

```
产出: Phase 2 | 写入: copilot | 读取: copilot (后续章节), formatter (Phase 3)
路径: 05_撰写过程/_引用编号注册表.md
```

```markdown
# 引用编号注册表

> 项目: {project_name} | 最后更新: YYYY-MM-DD
> 规则: 编号 = 首次出现顺序，全文唯一，不可重用

## 编号分配

| 编号 | cite_key | 首次出现 | 引用类型 | 备注 |
|------|----------|---------|---------|------|
| [1] | Patterson_2017 | 第1章 | 背景引用 | 计算机体系结构经典教材 |
| [2] | Waterman_2019 | 第1章 | 背景引用 | RISC-V 规范 |
| [3] | Bao_2021 | 第2章 | 方法引用 | 一生一芯训练体系 |
| [4] | Harris_2021 | 第2章 | 方法引用 | 数字电路设计方法 |
| [5] | Shukla_2023 | 第2章 | 对比引用 | FPGA vs ASIC 对比 |
| ... | ... | ... | ... | ... |

## 编号使用统计

| 章节 | 引用数量 | 编号范围 |
|------|---------|---------|
| 第1章 | 2 | [1]-[2] |
| 第2章 | 3 | [3]-[5] |
| 第3章 | 5 | [6]-[10] |
| ... | ... | ... |

## 交叉引用

| 被引用元素 | 类型 | 定义位置 | 被引用位置 |
|-----------|------|---------|-----------|
| 图3-2 数据通路 | 图 | 第3章 §3.2 | 第5章 §5.1 |
| 表2-1 指令编码 | 表 | 第2章 §2.3 | 第3章 §3.1 |
```

### 规则

- **编号一旦分配不可更改**。新增引用从最大编号+1 开始
- cite_key 必须与 `04_参考文献/literature_cards/{cite_key}.md` 的文件名完全一致
- 「首次出现」标注章节名（如"第3章"），不必标注节号
- copilot 每完成一章后，必须追加该章的引用到此表

## Schema 5: 文献索引卡

```
产出: Phase 1a | 写入: literature-agent | 读取: copilot, formatter
路径: 04_参考文献/literature_cards/{第一作者姓氏}_{年份}.md
```

```markdown
# {论文标题}

| 字段 | 值 |
|------|-----|
| **作者** | {完整作者列表，英文用原名} |
| **年份** | {YYYY} |
| **来源** | {期刊全称 / 会议全称 / arXiv} |
| **DOI/URL** | {https://doi.org/...} |
| **cite_key** | {作者}_{年份} |
| **关键词** | {关键词1}, {关键词2}, {关键词3} |

## 核心发现

{200-300 字}
研究问题: ...
方法: ...
主要发现: ...
结论: ...

## 研究方法

{100-200 字}
实验设计: ...
数据来源: ...
样本量: ...
分析方法: ...

## 关键摘录

> {直接引用的原文段落} — 第{页码}页

## 与本文关系

将被引用在: 第{N}章（{用途说明}）
引用类型: 背景引用 / 方法引用 / 对比引用 / 数据引用
```

### 规则

- DOI/URL 必须可点击（完整的 https:// 链接）
- 「关键摘录」为可选字段
- 「与本文关系」在 Phase 1b 骨架确认后补充
- 文件命名为 `{第一作者英文姓氏}_{年份}.md`，如 `Patterson_2017.md`

## 通用规则

1. **所有文件第一行为 `# 标题`**，不以 frontmatter 开头（订单摘要除外）
2. **日期格式统一为 `YYYY-MM-DD`**，时间格式 `HH:MM`
3. **表格用 Markdown 原生语法**，不用 HTML table
4. **状态标记用 emoji**: ✅ 通过 / ⚠️ 警告 / ❌ 失败
5. **Agent 写入时用实际数据替换所有 `{占位符}`**
